What this system actually does

It evolves on purpose

When it spots a gap, it sends a focused agent, captures the success, and turns that into a reusable skill. The skill list lives in skills/INDEX.json.

It runs with guardrails

Benchmarks can be segmented. Memory can switch to safe mode. The goal is stable output without hammering your SSD, VRAM, or CPU.

It stays local by design

Embedding backends prioritize NPU, then Ollama, then CPU. That keeps memory fast and avoids unnecessary downloads.

How the evolution loop feels day to day

Step 1

Hunt the gap

GPIA detects a missing capability and triggers a focused exploration.

Step 2

Spawn and solve

Ephemeral agents handle the task with the right model for the job.

Step 3

Dissect and synthesize

What worked is distilled into a new skill, which becomes permanent.

Hardware-aware defaults

Embedding order

  • NPU (OpenVINO)
  • Ollama embeddings
  • sentence-transformers (CPU)

Default Ollama model: mahonzhan/all-MiniLM-L6-v2 (GPIA alias: gpia-minilm-l6-v2).

Safer memory runs

python gpia_benchmark_suite.py --sections memory --memory-mode safe --guardrails on

Safe mode reduces disk churn and avoids heavy embedding work.

Segmented benchmarks

python gpia_benchmark_suite.py --sections model,s2,memory --guardrails on

Smaller bursts keep CPU, GPU, and SSD usage within safer ranges.
